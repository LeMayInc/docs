LeMay Commerce exposes 129 discrete tools, each mapped to a specific Shopify Admin GraphQL operation. Tools are organized into 24 categories that correspond to the fundamental domains of commerce operations.

## Tool Availability by Tier

Not every tier has access to every tool. The table below maps subscription levels to tool counts. Higher tiers unlock progressively more specialized operations — automation workflows, bulk mutations, and advanced analytics.

| Tier | Tools | Categories |
|------|-------|------------|
| Startup | 65 | 12 core categories |
| Small Business | 80 | 16 categories |
| Business | 105 | 20 categories |
| Corporate | 129 | All 24 categories |
| Enterprise | 129 | All 24 + automation workflows |
| LeMay Agency | 129 | All 24 + dedicated account manager |

## Tool Anatomy

Every tool in LeMay Commerce follows the same structural pattern:

```typescript
{
  name: "orders_list",           // Unique tool identifier
  category: "orders",            // Parent category
  description: "List orders...", // Natural language description
  inputSchema: { ... },         // JSON Schema for parameters
  handler: async (params) => {  // Execution function
    // 1. Validate parameters
    // 2. Construct GraphQL query
    // 3. Execute against Shopify Admin API
    // 4. Return structured response
  }
}
```

The AI client sees the tool name, description, and input schema. When it selects a tool and provides parameters, the handler executes the corresponding GraphQL operation and returns the result. There is no intermediate interpretation layer.
