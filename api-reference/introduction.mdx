LeMay Commerce exposes its functionality through the [Model Context Protocol](https://modelcontextprotocol.io) (MCP). There is no REST API, no webhook-based integration, and no SDK to install beyond the MCP server itself.

## How It Works

Your AI client connects to the LeMay Commerce MCP server, which provides a manifest of 129 tools. The AI client selects tools based on your natural language input and invokes them with structured parameters. The MCP server executes the corresponding Shopify GraphQL operation and returns the result.

```
AI Client  ←→  MCP Protocol  ←→  LeMay Commerce Server  ←→  Shopify Admin API
```

## Connection

The MCP server is distributed as an npm package. Configure your AI client to launch it with your store credentials:

```json
{
  "mcpServers": {
    "lemay-commerce": {
      "command": "npx",
      "args": ["-y", "lemay-commerce"],
      "env": {
        "SHOPIFY_STORE": "your-store.myshopify.com",
        "SHOPIFY_ACCESS_TOKEN": "your-access-token"
      }
    }
  }
}
```

## Tool Schema

Every tool follows the MCP tool specification:

```typescript
interface Tool {
  name: string;          // Unique identifier (e.g., "orders_list")
  description: string;   // Natural language description
  inputSchema: {         // JSON Schema for parameters
    type: "object";
    properties: Record<string, JsonSchema>;
    required?: string[];
  };
}
```

When invoked, the server returns a structured response conforming to the MCP content format.

## Error Handling

Tool invocations return errors in the MCP standard format:

| Error Type | Description |
|-----------|-------------|
| `INVALID_PARAMS` | Required parameters missing or malformed |
| `SHOPIFY_API_ERROR` | Shopify Admin API returned an error |
| `PERMISSION_DENIED` | Tool not available on current subscription tier |
| `RATE_LIMITED` | Shopify API rate limit exceeded (automatic retry) |
